## 一、项目概述与现状分析

### 1.1 项目背景

本项目为「管制员学员扇区能力评估系统」微信小程序，面向航空管制培训场景，支持：
- **学员**查看自己的评分和成长轨迹
- **教员**对学员进行扇区能力评分
- **管理人员**进行用户管理、扇区配置、数据查看与分析

### 1.2 现有页面结构

| 页面 | 功能 | 主要用户 |
|------|------|---------|
| `login` | 用户登录（默认密码 `hdkg2007`） | 全部 |
| `radar` | 学员详情/能力雷达图 | 学员、教员 |
| `trend` | 成长趋势/学员列表 | 教员、管理 |
| `mygrades` | 成绩列表 | 学员、教员 |
| `score` | 评分工作 | 教员 |
| `overview` | 管理首页/评估记录 | 管理 |
| `analysis` | 数据分析/对比 | 管理 |
| `users` | 用户管理（含照片/性别/出生日期/等级/进组时间录入，年龄自动计算） | 管理 |
| `config` | 扇区配置 | 管理 |
| `detail` | 历史详情 | 全部 |
| `instructor-history` | 教员评分历史 | 教员 |
| `profile` | 个人资料展示与编辑（新增） | 全部 |

### 1.3 当前核心问题诊断

#### 视觉层面
- 深色主题层级不清，卡片边界模糊
- 颜色浑浊，状态色饱和度不统一
- 文字色阶不足，辅助信息辨识度低

#### 交互层面
- 数据是「死的」：看到学员不能立即评分，看到分数不能看趋势
- 评分页无历史参考，凭印象打分
- 无全局统一的点击反馈和退出保护

#### 功能层面
- 批量评分功能与实际业务不符，评分应个性化
- ICAO/体检提醒已有基础，但视觉表现力不足
- 缺少「待关注学员」等主动提醒机制

#### 架构层面
- 权限处理不统一，部分页面隐藏、部分禁用，规则不一致
- 选择器体验待优化（扇区选择、学员选择）

## 二、设计哲学与核心原则

### 2.1 简洁三原则

```
1. 一眼看懂：每个页面进入后 3 秒内明白能做什么
2. 一步到达：核心功能不超过 2 次点击
3. 一次说完：信息不重复，标签不堆砌
```

### 2.2 核心设计原则

| 原则 | 说明 |
|------|------|
| **深色精致化** | 保留航空管制专业感，建立 4 层背景层级系统 |
| **数据活态化** | 任何展示数据的地方都能触发操作，任何操作的地方都能参考数据 |
| **角色差异化** | 学员看成长、教员看评估、管理看对比，信息按需呈现 |
| **反馈即时化** | 100ms 内给出点击反馈，任何操作都有结果告知 |
| **退出安全化** | 未保存有保护、不可逆有确认、随时能回退 |

### 2.3 「学员→扇区→展示」核心逻辑

坚守现有信息架构，不推翻重造：
- **学员**是评估的主体
- **扇区**是评估的场景
- **展示**是评估的结果

优化方向：**让选择更顺畅，让展示更充分，让操作更快捷**。

## 三、深色主题精致化设计系统

### 3.1 当前深色主题的问题

| 问题 | 现状 | 影响 |
|------|------|------|
| 颜色层级只有2级 | 背景 `#0a1628` → 卡片 `#132238` | 信息层级扁平 |
| 卡片边框与背景对比度过低 | 边框 `#1e3a5f` 与卡片接近 | 卡片边界模糊 |
| 阴影在深色下失效 | `rgba(0,0,0,0.3)` 看不见 | 无法建立层级 |
| 状态色饱和度不一 | 绿 `#00d26a` 太亮，红 `#ff4d4f` 偏粉 | 视觉噪音大 |

### 3.2 精致化深色设计令牌

```css
/* ========== 背景层级 ========== */
--bg-page:       #070d17;   /* 页面底色（最深） */
--bg-surface-1:  #0f1724;   /* 卡片背景 */
--bg-surface-2:  #162033;   /* 浮层、展开项、hover */
--bg-surface-3:  #1e2d42;   /* 选中态、聚焦态 */

/* ========== 边框与分割 ========== */
--border-subtle: #1a2740;   /* 卡片默认边框 */
--border-active: #3b82f6;   /* 聚焦、选中边框 */
--divider:       #1a2740;   /* 列表分割线 */

/* ========== 文字色阶 ========== */
--text-primary:   #e2e8f0;   /* 标题、正文 */
--text-secondary: #94a3b8;   /* 辅助说明 */
--text-muted:     #475569;   /* placeholder、禁用 */

/* ========== 状态色 ========== */
--accent-blue:   #3b82f6;   /* 主按钮、链接 */
--accent-cyan:   #22d3ee;   /* 次要强调 */
--success:       #22c55e;   /* 通过、合格 */
--warning:       #f59e0b;   /* 警告、即将到期 */
--danger:        #ef4444;   /* 不合格、删除 */

/* ========== 光晕效果 ========== */
--glow-blue:  0 0 16rpx rgba(59, 130, 246, 0.15);
--glow-green: 0 0 16rpx rgba(34, 197, 94, 0.15);
--glow-red:   0 0 16rpx rgba(239, 68, 68, 0.15);
```

### 3.3 层级建立规则

```
页面背景     : #070d17（最底层）
  └── 卡片   : #0f1724（+1 层，border: 1px solid #1a2740）
      └── 卡片内展开项 : #162033（+2 层）
          └── 选中/聚焦 : #1e2d42（+3 层，border: 1px solid #3b82f6）
```

**关键原则**：深色主题下，**用「边框亮度」和「背景亮度差异」替代「阴影」建立层级**。

## 四、权限交互统一规范

### 4.1 三级权限处理模型

| 级别 | 策略 | 处理方式 | 适用场景 |
|------|------|---------|---------|
| **Level A：隐藏** | 完全不渲染 | 元素从 DOM 中移除 | TabBar 导航、页面入口、管理功能入口 |
| **Level B：禁用** | 灰色显示，点击提示原因 | `opacity: 0.3` + `grayscale(0.8)`，点击弹出「您无权限进行此操作」 | 页面内操作按钮（删除、放单、提交） |
| **Level C：降级** | 自动过滤数据 | 不显示不属于当前用户的数据 | 学员只能看自己的成绩、教员只能看自己科室的学员 |

### 4.2 各角色权限矩阵

| 功能 | 学员 | 教员 | 科室副主任 | 科室主任 | 培训主管 | 中心主任 |
|------|------|------|-----------|---------|---------|---------|
| 查看自己成绩 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 查看所有学员成绩 | ❌ | ✅(本科室) | ✅(本科室) | ✅(本科室) | ✅(本科室) | ✅(全中心) |
| 评分 | ❌ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 用户管理 | ❌ | ❌ | ✅(本科室) | ✅(本科室) | ✅(本科室) | ✅(全中心) |
| 扇区配置 | ❌ | ❌ | ✅(本科室) | ✅(本科室) | ✅(本科室) | ✅(全中心) |
| 评分任务配置 | ❌ | ❌ | ✅(本科室) | ✅(本科室) | ✅(本科室) | ✅(全中心) |
| 数据分析 | ❌ | ❌ | ✅(本科室) | ✅(本科室) | ✅(本科室) | ✅(全中心) |
| 放单操作 | ❌ | ❌ | ✅(本科室) | ✅(本科室) | ✅(本科室) | ✅(全中心) |

### 4.3 禁用态统一样式

```css
.btn-disabled {
  opacity: 0.3;
  filter: grayscale(0.8);
  pointer-events: none;
}
/* 或保留点击事件用于提示 */
.btn-disabled-hint {
  opacity: 0.4;
  filter: grayscale(0.8);
}
.btn-disabled-hint:active {
  opacity: 0.4; /* 不变化，暗示不可点 */
}
```

### 4.4 权限工具函数

**新建 `utils/permission.js`**：

```javascript
const { MANAGER_ROLES, isInstructorRole } = require('./roles.js');

/**
 * 判断当前用户是否能管理目标用户
 * 中心主任：可管理任意科室
 * 科室副主任/科室主任/培训主管：仅能管理本科室
 */
function canManageUser(currentUser, targetUser) {
  if (!currentUser || !targetUser) return false;
  if (currentUser.role === 'center_director') return true;
  if (['deputy_director', 'department_head', 'supervisor'].includes(currentUser.role)) {
    return targetUser.department === currentUser.department;
  }
  return false;
}

/**
 * 判断当前用户是否能查看目标学员的成绩
 * 中心主任：可查看任意科室
 * 科室副主任/科室主任/培训主管/教员：仅能查看本科室
 */
function canViewStudentScores(currentUser, student) {
  if (!currentUser || !student) return false;
  if (currentUser.userId === student.userId) return true;
  if (currentUser.role === 'center_director') return true;
  if (isInstructorRole(currentUser.role) || ['deputy_director', 'department_head', 'supervisor'].includes(currentUser.role)) {
    return student.department === currentUser.department;
  }
  return false;
}

module.exports = { canManageUser, canViewStudentScores };
```

## 五、学员→扇区→展示逻辑优化

### 5.1 核心逻辑坚守

不推翻现有「先选学员/扇区，再展示详情」的信息架构，只优化选择器的交互体验。

### 5.2 扇区选择器优化：Pill Tabs

将扇区选择从系统 picker 改为横向滚动的 Pill 标签：

```
┌─────────────────────────────────────────┐
│ ◀ ACC02/32  ACC08  ACC18/28  ACC36  ▶  │
│    └────── 选中态（蓝色背景+光晕）       │
└─────────────────────────────────────────┘
```

**优势**：
- 一眼看到所有可选扇区
- 切换扇区只需一次点击
- 选中态有明确视觉反馈

### 5.3 学员选择器优化：底部搜索面板

将学员选择从系统 picker 改为底部滑出面板：

```
┌─────────────────────────────────────────┐
│ 选择学员                    [✕]          │
│ ┌─────────────────────────────────────┐ │
│ │ 🔍 搜索学员姓名...                   │ │
│ └─────────────────────────────────────┘ │
│ 张小明  区域一室  初阶一段          ✅   │
│ 李华    区域一室  初阶二段              │
│ 王五    区域二室  初阶一段              │
│ ...                                     │
└─────────────────────────────────────────┘
```

**优势**：
- 支持搜索，学员多时快速定位
- 可显示学员更多信息（科室、阶段）
- 选中态即时反馈

### 5.4 记忆上次选择

- 本地存储 `lastSelectedSectorId` 和 `lastSelectedStudentId`
- 刷新页面后自动恢复上次选择
- 切换页面返回后保持选择状态

### 5.5 切换过渡动画

切换扇区或学员时，增加淡入淡出过渡：
```css
.fade-transition {
  animation: fadeIn 0.3s ease;
}
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(10rpx); }
  to { opacity: 1; transform: translateY(0); }
}
```

## 六、活数据联动设计

### 6.1 核心原则

```
任何展示数据的地方 → 都可以触发相关操作
任何操作的地方 → 都可以参考相关数据
```

### 6.2 具体联动场景矩阵

| 场景 | 数据展示 | 可触发操作 | 目标页面 |
|------|---------|-----------|---------|
| **趋势页学员卡片** | 姓名、最新分数、趋势 | 查看历史趋势 / 去评分 | radar / score |
| **评分页顶部** | 当前学员+扇区 | 查看该学员该扇区历史参考 | 本页面板 |
| **成绩列表** | 分数、等级 | 展开雷达图详情 / 查看进步对比 | detail |
| **管理页记录** | 学员、教员、分数 | 点击学员名 → 学员详情 / 点击教员名 → 教员历史 | student-profile / instructor-history |
| **雷达图** | 维度得分 | 点击维度 → 该维度历史趋势 | trend-popup |

### 6.3 页面跳转参数规范

```
# 跳转到评分页，自动选择学员和扇区
/pages/score/score?studentId=xxx&sectorId=xxx

# 跳转到雷达页，查看指定学员
/pages/radar/radar?studentId=xxx&sectorId=xxx

# 跳转到学员详情页
/pages/student-profile/student-profile?studentId=xxx

# 跳转到分析页，预筛选学员
/pages/analysis/analysis?studentIds=xxx,yyy&sectorId=xxx
```

### 6.4 数据联动网络图

```
   trend页 ──点击学员──→ 学员详情面板 ──去评分──→ score页
      │                      │
      │                      ↓
      │               历史趋势图(底部面板)
      │
      └──点击「去评分」──→ score页(自动带入学员+扇区)

   radar页 ──点击历史记录──→ 该次评分详情
      │
      └──切换扇区──→ 自动加载该扇区数据

   score页 ──顶部「查看历史」──→ 该学员历史面板
      │
      └──评分完成──→ 结果页 ──「查看趋势」──→ radar页

   analysis页 ──点击雷达图维度──→ 维度对比详情
      │
      └──点击学员名──→ 学员个人详情页
```

## 九、ICAO/体检全生命周期提醒体系

### 9.1 提醒规则精确定义

| 证件类型 | 有效期 | 提醒触发点 | 提醒频率 | 提醒对象 |
|---------|--------|-----------|---------|---------|
| **ICAO 考试日期** | 3 年 | 到期前 **6 个月** | 每月 1 次 + 到期后每周 1 次 | 本人 + 直属管理人员 |
| **体检日期** | **≤ 39 岁**: 2 年<br>**≥ 40 岁**: 1 年 | 到期前 **3 个月** | 每月 1 次 + 到期后每周 1 次 | 本人 + 直属管理人员 |

**体检有效期年龄计算**：
```
年龄 = 当前年份 - 出生年份（根据 birthDate 计算）
if (年龄 >= 40) 体检有效期 = 1 年
else 体检有效期 = 2 年
```

**状态分级**：
```
正常 (normal)     → 绿色，不提醒
即将到期 (warning) → 黄色，按月提醒
已过期 (expired)   → 红色，每周提醒，直到更新
```

### 9.2 提醒展示样式

|---------|---------|---------|---------|
| **评分即将到期** | 距离上次评分超过 `(间隔 - 1天)` | 每日 1 次 | 负责教员 + 直属管理人员 |
| **评分已逾期** | 超过间隔天数未评分 | 每 3 天 1 次，直到评分完成 | 负责教员 + 直属管理人员 + 中心主任 |
| **新学员待分配** | 新增学员 24 小时内无负责教员 | 即时 + 每日 1 次 | 管理人员 |

**状态分级**：
```
正常 (normal)       → 绿色，7 天内已完成评分
即将到期 (warning)  → 黄色，接近评分间隔期限
已逾期 (expired)    → 红色，超过间隔期限未评分
```

### 10.4 全局统一提醒中心

所有提醒（ICAO / 体检 / 评分任务）统一入口、统一样式、统一展示位置。

#### 10.4.1 提醒聚合展示规范

无论哪种提醒，均遵循以下统一结构：
```
┌─────────────────────────────────────────┐
│ ⚠ 提醒中心  共 6 项待处理                 │  ← 迷你提醒条（固定顶部）
│   🔴 评分逾期: 2人   🟡 评分即将到期: 3人  │

└─────────────────────────────────────────┘
```

- 按「已逾期 / 即将到期 / 正常」分组
- 同组内按紧急程度升序排列（最紧急在最上）
- 评分提醒点击 → 直接跳转到 `score` 页并带入学员+扇区
- ICAO/体检提醒点击 → 打开用户编辑弹窗
- 点击「已读」→ 本次会话内不再提醒（本地存储）

#### 10.4.2 各角色提醒展示位置

#### 10.4.3 教员端提醒样式

## 十一、功能精简与减法设计

### 11.1 去除批量评分

**原因**：
- 评分是高度个性化的，每个学员表现不同
- 批量评分容易导致「敷衍评分」，失去评估意义
- 与核心「学员→扇区→展示」逻辑不符

**移除范围**：
- `pages/batch/batch.wxml/js/wxss/json` 整页删除
- `app.json` pages 数组中删除 `"pages/batch/batch"`
- 全局搜索 `batch`，清理所有跳转链接

**替代方案**：趋势页学员卡片上的「去评分」按钮，一键跳转并自动带入学员+扇区。

### 11.2 减法原则

```
看不到的 = 不存在的
用一次的 = 可隐藏的
能合并的 = 不分开的
```

```javascript
function confirmDelete({ title = '确认删除', content = '此操作不可恢复，确定继续吗？', confirmText = '删除' } = {}) {
  return new Promise((resolve) => {
    wx.showModal({ title, content, confirmText, confirmColor: '#ff4d4f', success: (res) => resolve(res.confirm) });
  });
}

function confirmDiscard({ title = '放弃编辑', content = '有未保存的更改，确定放弃吗？' } = {}) {
  return new Promise((resolve) => {
    wx.showModal({ title, content, confirmText: '放弃', confirmColor: '#ff4d4f', cancelText: '继续编辑', success: (res) => resolve(res.confirm) });
  });
}

function confirmExit({ title = '确认退出', content = '评分数据将丢失，确定退出吗？' } = {}) {
  return new Promise((resolve) => {
    wx.showModal({ title, content, confirmText: '退出', confirmColor: '#ff4d4f', cancelText: '留在当前页', success: (res) => resolve(res.confirm) });
  });
}

module.exports = { confirmDelete, confirmDiscard, confirmExit };
```
